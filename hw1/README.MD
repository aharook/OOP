# Домашня робота №1.  Виявлення God-class

## 1 Опишіть характеристики "God Object" анти-патерну.
God Object - антипатерн, при якому в одному класі створено забагато методів, що перенавантажують його, обмежують та можуть заважати коректному функціонуванню класу.
Також він порушує принцип SRP також він має занадто багато залежностей від інших частин коду. Це погіршує не тільки функціонування коду, а й виявлення помилок, некоректних обчислень та подальшої підтримки цього коду. 

## 2 Наведіть приклад простого класу, який порушує SRP, і поясніть, чому.

    Class Student 
    {
        public:
        std::string name;
        double grade;

        void calculateGrade(){
            std::cout << "calculating..." << std::endl;
        }

        void printStudent() {
            std::cout << "Name: " name, "Grade: " grade << std::endl;
        }

        void savestudent() {
            std::cout << "saving student..." << std::endl;
        }
    };

Я навів найпростіший приклад God Object класу. Його основною проблемою є - забагато логіки для 1 класу, тому якщо зміниться будь-яка логіка, то довдеться міняти той самий клас. Саме це є порушенням принципів SRP, який вимагає лише одну відповідальність для 1 класу.
